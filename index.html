Does Artwork Actually Drive Clicks?

A simulated A/B test inspired by Netflix's real experimentation system — built in Python, analyzed with statistics, and translated into a business recommendation.

Western Washington University · College of Business and Economics

Context

The Business Problem

Netflix has over 200 million subscribers choosing from thousands of titles every day. The artwork that represents each title — the thumbnail image a user sees while browsing — is one of the strongest levers the company has to influence whether someone clicks "play" or keeps scrolling.

But which artwork works best? Designers and marketers can debate preferences endlessly. Netflix settles the question with data. The company runs thousands of A/B tests every year, showing different versions of artwork to different groups of users and measuring which version drives more engagement. This practice sits at the intersection of machine learning and experimentation: algorithms personalize artwork selection at scale, and rigorous testing validates that each change actually improves the experience.

This project replicates that logic. I simulated a controlled experiment between two artwork versions, measured the results, ran a statistical test, and arrived at a recommendation — the same workflow a Netflix data science team would follow before shipping a change to production.

Methodology

What I Built

The simulation runs in three steps, each building on the last. Together they form a complete experimentation pipeline — from data generation through statistical analysis to a go/no-go decision.

1

Simulate the Experiment

I created 1,000 virtual users and randomly assigned each one to see either Artwork A or Artwork B. Users who saw Artwork A had a 24% probability of clicking; users who saw Artwork B had a 30% probability. A random number generator simulated whether each user actually clicked, producing a dataset that mirrors what Netflix's event logging pipeline would capture in a real test.

2

Measure the Results

I grouped the data by artwork version and calculated three metrics for each group: total users, total clicks, and the click-through rate (CTR). These are the first numbers a product team checks when an experiment closes — they reveal whether one version is outperforming the other and by how much.

3

Test for Statistical Significance

A higher click rate in the sample doesn't automatically mean the artwork is better — the gap could be random noise. I ran a two-proportion z-test to calculate the probability that a difference this large would appear if the two artworks were actually identical. If that probability (the p-value) falls below 5%, the result is considered statistically significant — meaning the difference is almost certainly real, not luck.

Reading the Results

Artwork B drove roughly 30% of users to click, compared to about 24% for Artwork A — a lift of approximately 6 percentage points. The z-test returned a p-value well below the standard 0.05 threshold, which means we can be at least 95% confident this gap reflects a genuine difference in user behavior, not random chance in the sample.

In practical terms: if Netflix showed Artwork B to its entire user base instead of Artwork A, the data strongly suggests more people would engage with the title.

Decision

Business Recommendation

Ship Artwork B

The experiment produced a statistically significant result: Artwork B outperforms Artwork A by a meaningful margin. Based on this evidence, the recommendation is to roll out Artwork B to all users for this title. In a production environment, Netflix would log this decision, archive the experiment data, and feed the result back into its machine learning personalization models so the system gets smarter with every test it runs.

This is exactly how experimentation drives value in a data-driven organization. Instead of relying on intuition or committee debates, the company lets user behavior answer the question — and the answer is clear.

Implementation

Complete Python Code

The full simulation, analysis, and visualization — ready to run in Google Colab or any Python environment with NumPy, pandas, matplotlib, and statsmodels installed.

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from statsmodels.stats.proportion import proportions_ztest

# ==================================================================
# STEP 1: SIMULATE THE EXPERIMENT
# ==================================================================

# Lock in randomness so results are identical on every run
np.random.seed(42)

# ------------------------------------------------------------------
# Define experiment parameters
#   - 1,000 total users in the test
#   - Artwork A: 24% true click probability
#   - Artwork B: 30% true click probability
# ------------------------------------------------------------------
n_users = 1000
click_prob_a = 0.24
click_prob_b = 0.30

# Create sequential user IDs from 1 through 1,000
user_ids = np.arange(1, n_users + 1)

# Randomly assign each user to see Artwork A or Artwork B (50/50 split)
artwork_versions = np.random.choice(['A', 'B'], size=n_users)

# ------------------------------------------------------------------
# Simulate clicks using a Bernoulli trial for each user
#   - np.where picks the correct click probability based on group
#   - np.random.binomial(1, p) returns 1 (clicked) with probability p
# ------------------------------------------------------------------
click_probs = np.where(artwork_versions == 'A', click_prob_a, click_prob_b)
clicks = np.random.binomial(1, click_probs)

# Assemble the three columns into a single DataFrame
df = pd.DataFrame({
    'user_id': user_ids,
    'artwork': artwork_versions,
    'clicked': clicks
})

# ==================================================================
# STEP 2: CALCULATE GROUP-LEVEL METRICS
# ==================================================================

# ------------------------------------------------------------------
# Group the DataFrame by artwork version and compute three metrics:
#   - total_users: how many users were assigned to each group
#   - total_clicks: how many of those users clicked
#   - click_through_rate: total_clicks / total_users
# ------------------------------------------------------------------
summary = df.groupby('artwork')['clicked'].agg(
    total_users='count',
    total_clicks='sum'
)

# Compute click-through rate as a new column (clicks / users)
summary['click_through_rate'] = summary['total_clicks'] / summary['total_users']

# Print the summary table
print("=" * 55)
print("GROUP-LEVEL SUMMARY")
print("=" * 55)
print(summary)
print()

for group_name, row in summary.iterrows():
    print(f"Artwork {group_name}: "
          f"{row['total_users']:.0f} users, "
          f"{row['total_clicks']:.0f} clicks, "
          f"CTR = {row['click_through_rate']:.2%}")

# ==================================================================
# STEP 3: VISUALIZE THE CLICK-THROUGH RATES
# ==================================================================

fig, ax = plt.subplots(figsize=(6, 4))

groups = summary.index.tolist()
ctrs = summary['click_through_rate'].tolist()
colors = ['#4A90D9', '#E85D75']

bars = ax.bar(groups, ctrs, color=colors, width=0.5, edgecolor='white')

# Add the exact CTR percentage on top of each bar
for bar, ctr in zip(bars, ctrs):
    ax.text(
        bar.get_x() + bar.get_width() / 2,
        bar.get_height() + 0.005,
        f"{ctr:.2%}",
        ha='center', fontsize=13, fontweight='bold'
    )

ax.set_title('Click-Through Rate by Artwork Version', fontsize=14, fontweight='bold')
ax.set_xlabel('Artwork Version', fontsize=12)
ax.set_ylabel('Click-Through Rate', fontsize=12)
ax.set_ylim(0, max(ctrs) + 0.08)
ax.yaxis.set_major_formatter(plt.FuncFormatter(lambda y, _: f'{y:.0%}'))
ax.spines['top'].set_visible(False)
ax.spines['right'].set_visible(False)
ax.grid(axis='y', alpha=0.3)

plt.tight_layout()
plt.show()

# ==================================================================
# STEP 4: STATISTICAL SIGNIFICANCE TEST
# ==================================================================

clicks_a = summary.loc['A', 'total_clicks']
clicks_b = summary.loc['B', 'total_clicks']
n_a = summary.loc['A', 'total_users']
n_b = summary.loc['B', 'total_users']

# Run a two-proportion z-test
#   - count: clicks per group
#   - nobs: users per group
#   - alternative='two-sided': test for any difference
z_stat, p_value = proportions_ztest(
    count=np.array([clicks_a, clicks_b]),
    nobs=np.array([n_a, n_b]),
    alternative='two-sided'
)

alpha = 0.05

print()
print("=" * 55)
print("STATISTICAL SIGNIFICANCE TEST (Two-Proportion Z-Test)")
print("=" * 55)
print(f"  Artwork A CTR:   {summary.loc['A', 'click_through_rate']:.2%}")
print(f"  Artwork B CTR:   {summary.loc['B', 'click_through_rate']:.2%}")
print(f"  Observed lift:   {summary.loc['B', 'click_through_rate'] - summary.loc['A', 'click_through_rate']:.2%}")
print(f"  Z-statistic:     {z_stat:.4f}")
print(f"  P-value:         {p_value:.4f}")
print(f"  Significance α:  {alpha}")
print("-" * 55)

if p_value < alpha:
    print("  ✅ RESULT: Statistically significant difference detected.")
    print("     The gap in click-through rates is very unlikely to be")
    print("     random noise. Artwork B is the stronger performer.")
else:
    print("  ⚠️  RESULT: No statistically significant difference.")
    print("     We cannot confidently say the artworks perform differently.")
    print("     The observed gap could plausibly be due to chance.")
print("=" * 55)
